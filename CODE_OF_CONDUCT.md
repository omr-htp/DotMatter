# Code of Conduct

## Our Pledge

We want DotMatter to be an open, respectful, and technically rigorous project.
Contributors, maintainers, and participants are expected to engage in a way that
keeps the project constructive, professional, and safe for everyone.

## Expected Behavior

- Be respectful in technical disagreement.
- Focus feedback on code, design, behavior, and evidence.
- Assume good intent, but be precise when calling out problems.
- Accept and act on constructive feedback.
- Help keep discussions relevant to the project.

## Unacceptable Behavior

- Harassment, intimidation, or discriminatory language.
- Personal attacks, insults, or bad-faith provocation.
- Publishing private information without explicit permission.
- Disruptive behavior that repeatedly derails technical work.
- Any conduct that a maintainer reasonably determines is inappropriate for the project.

## Maintainer Responsibilities

Maintainers may edit, remove, or reject comments, commits, issues, pull requests,
and other contributions that violate this Code of Conduct. They may also
temporarily or permanently restrict participation when needed to protect the project.

## Scope

This Code of Conduct applies in project spaces, including issues, pull requests,
discussions, code review, and any other communication channels used for DotMatter work.

## Reporting

Report Code of Conduct concerns to the project maintainers through a private channel
that is appropriate for the repository host. Reports will be reviewed promptly and handled
with reasonable confidentiality.

## Enforcement

Maintainers will review reported incidents, determine an appropriate response, and
apply enforcement consistently based on the severity and context of the behavior.
