# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in DotMatter, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please use GitHub's [private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability) feature on this repository.

We will acknowledge your report within 48 hours and provide a timeline for a fix.

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest `main` | ✅ |

## Scope

This policy covers the DotMatter .NET libraries and controller service. Third-party dependencies (BouncyCastle, mDNS) should be reported to their respective maintainers.
